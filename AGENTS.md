# AGENTS.md

## プロジェクト概要

このプロジェクトは、Windows 11向けの軽量動画プレーヤー **UltraLight TC Player for Windows（仮）** を開発するためのものです。

目的は、一般的な動画プレーヤーでは不足しがちな **タイムコード確認**、**フレーム単位の操作**、**IN / OUT取得とffmpeg切り出しコマンド生成** を、軽量なWindowsネイティブアプリとして実現することです。

## Codexへの基本指示

Codexは、以下の優先順位で実装・提案・修正を行ってください。

1. 軽量性
2. H.264 mp4再生の安定性
3. IN / OUT取得とffmpegコマンド生成の実用性
4. タイムコード表示の視認性
5. UIの美しさ

UIの完成度よりも、まずは **動画を開ける・再生できる・タイムコードを表示できる・左右キーでフレーム移動できる・IN / OUTを記録できる・ffmpegコマンドをコピーできる** ことを優先してください。

## MVPのゴール

MVPでは、以下を完了条件とします。

- Windows 11で起動できる。
- 動画ファイルを開ける。
- `.mp4` / `.mov` / `.m4v` の再生を想定する。
- H.264のmp4動画を安定して再生できる。
- `Space` で再生 / 一時停止できる。
- `Right Arrow` で1フレーム送りができる。
- `Left Arrow` で1フレーム戻しができる。
- `hh:mm:ss:ff` のタイムコードを常時表示できる。
- FPSを自動判別し、タイムコード計算とフレーム移動に反映できる。
- `I` / `O` でIN / OUTを記録できる。
- `Esc` で選択中のINまたはOUTをクリアできる。
- `Delete` でIN / OUTを両方クリアできる。
- `Clear In` / `Clear Out` / `Clear All` ボタンがある。
- シークバー、音量スライダー、ミュートボタンがある。
- IN / OUTが揃った時だけffmpegコマンドを生成し、コピーできる。
- 出力ファイル名は再生中のソースファイル名から `source-cut.ext` として動的に生成する。
- UIはシンプルで、動画確認に集中できる。

## 実装時の重要ポイント

### タイムコード

- 表示形式は `hh:mm:ss:ff` とします。
- FPSは動画ファイルのメタデータから取得してください。
- 23.976 / 24 / 25 / 29.97 / 30 / 59.94 / 60 fps を想定してください。
- 初期版では、29.97 / 59.94 fps のドロップフレーム表記は必須ではありません。
- VFR（可変フレームレート）は初期版では厳密対応不要ですが、READMEなどに制限として明記してください。
- ffmpegコマンドでは、同じ位置を `hh:mm:ss.fff` に変換して使用してください。

### フレーム操作

- 1フレームの秒数は `1 / fps` として計算してください。
- フレーム送り / 戻し時は、現在位置から1フレーム分だけシークしてください。
- フレーム送り / 戻しは厳密なデコードフレーム単位ではなく、目安操作で構いません。
- 安定性を優先する場合、フレーム操作時は一時停止してから移動して構いません。

### IN / OUT とffmpeg

- `I` キーで現在位置をINに設定してください。
- `O` キーで現在位置をOUTに設定してください。
- `Esc` キーで選択中のINまたはOUTだけをクリアしてください。
- `Delete` キーでIN / OUTを両方クリアしてください。
- UIには `Clear In` / `Clear Out` / `Clear All` ボタンを置いてください。
- コマンド欄は常時表示し、IN / OUTが揃った時だけコピー可能にしてください。
- 入力ファイルは現在再生しているソースファイルのフルパスを使用してください。
- 出力ファイル名は同じフォルダに `source-cut.ext` 形式で生成してください。
- MVPでは `ffmpeg -ss ... -to ... -i "source" -c copy "output"` 形式のコマンドを生成してください。
- MVPではアプリ内でffmpegを実行せず、コマンドのコピーまでで構いません。

### UI

- 映像表示領域を主役にしてください。
- コントロールは最小限にしてください。
- フローティングコントロールは理想ですが、MVPでは固定表示でも構いません。
- タイムコード表示は横揺れしない等幅数字・等幅フォントを使ってください。

候補フォント:

- Cascadia Mono
- Consolas
- Segoe UIの等幅数字指定が可能な場合はそれを使用

## 重要な判断基準

このプロジェクトは、汎用動画プレーヤーではなく、**自分専用の軽量タイムコード確認・IN / OUT取得ツール** です。

そのため、機能を増やすよりも、以下を優先してください。

- 起動が速い
- 操作が軽い
- タイムコードが見やすい
- フレーム単位で確認できる
- IN / OUTをすばやく取得できる
- ffmpegコマンドを迷わずコピーできる
- 余計なUIがない
